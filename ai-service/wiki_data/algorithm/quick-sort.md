# 快速排序（Quick Sort）

## 一、定义
快速排序是一种基于分治思想的高效排序算法。通过选择一个基准元素（pivot），将数组划分为小于基准和大于基准的两部分，然后递归地对两部分分别排序，最终整体有序。

## 二、核心原理
1. **选择基准**：从数组中选取一个元素作为 pivot（可选首元素、尾元素、中间元素或随机）
2. **分区操作（Partition）**： rearrange 数组，使 pivot 左边都小于等于它，右边都大于它
3. **递归排序**：对 pivot 左右两个子数组分别递归执行快速排序
4. **终止条件**：子数组长度为 0 或 1 时自然有序，停止递归

## 三、算法过程
以数组 `[6, 2, 8, 5, 1, 9, 3]` 为例，选最后一个元素 3 作为 pivot：

**第一轮分区**：
- 遍历数组，将小于 3 的放左边，大于 3 的放右边
- 分区结果：`[2, 1, 3, 5, 8, 9, 6]`
- pivot 3 已经在最终正确位置

**递归处理左半部分 `[2, 1]`**：
- 选 1 为 pivot，分区后 `[1, 2]`，完成

**递归处理右半部分 `[5, 8, 9, 6]`**：
- 选 6 为 pivot，分区后 `[5, 6, 9, 8]`
- 继续递归 `[5]` 和 `[9, 8]`，最终全部有序

最终结果：`[1, 2, 3, 5, 6, 8, 9]`

## 四、复杂度分析
- **时间复杂度**：
  - 平均情况：O(n log n)，每次分区大致平分
  - 最好情况：O(n log n)，每次均匀划分
  - 最坏情况：O(n²)，数组已有序且选端点为 pivot 时退化为冒泡
- **空间复杂度**：O(log n)，递归调用栈的深度
- **稳定性**：不稳定排序，分区过程中相等元素的相对顺序可能改变

## 五、常见误区
1. **最坏情况忽视**：如果数组接近有序且总选端点做 pivot，会退化到 O(n²)，实际工程中常用随机选 pivot 或三数取中避免
2. **分区边界写错**：双指针分区时左右指针的移动顺序、交换条件容易写错，导致死循环或分区错误
3. **递归终止条件遗漏**：忘记处理 `left >= right` 的边界，会导致栈溢出
4. **混淆与归并排序**：快排是「先分区再递归」，归并是「先递归再合并」，两者分治方向相反
